## 1. Цель сайта

### Задача

Лендинг — главный инструмент привлечения клиентов. Сайт должен за 5–10 секунд ответить на вопрос «кто это и чем он полезен мне» и побудить написать или оставить заявку.

### Целевое действие пользователя

Приоритет 1 — **написать в Telegram** (кнопка в Hero и floating-кнопка на мобильном)
Приоритет 2 — **заполнить форму** в секции «Контакты» (3 поля)

Конверсия считается состоявшейся, когда пользователь отправил сообщение или форму.

### Метрики успеха MVP

| Метрика | Цель |
|---|---|
| PageSpeed Insights | > 85 |
| Bounce rate | < 55% |
| Scroll depth | > 65% |
| Конверсия в заявку | > 3% |

## 2. Целевая аудитория

### Основной клиент

**Стартапы и продуктовые команды**, которым нужен технический партнёр для запуска или развития цифрового продукта.

- Нужен MVP, SaaS, приложение или AI-инструмент
- Ценят скорость и прямую коммуникацию
- Устали от агентств — медленно, дорого, бюрократия
- Технически грамотны или хотят разобраться

### Вторичный клиент

Малый бизнес, которому нужна автоматизация через AI-агентов или быстрый сайт.

### Главные вопросы клиента при заходе на сайт

1. «Он уже делал что-то похожее?» → закрывает секция «Услуги» + «AI-экспертиза»
2. «Сколько стоит и сколько времени?» → закрывает FAQ
3. «Почему он, а не агентство?» → закрывает секция «Почему со мной»
4. «Что такое AI-агент и зачем мне это?» → закрывает секция «AI-экспертиза»
5. «Как с ним работать?» → закрывает секция «Процесс работы»

### Что заставляет уйти

- Нет понимания за 5 секунд, чем этот человек полезен
- Медленная загрузка или кривая мобильная версия
- Только email как контакт (нет Telegram)
- Общие слова без конкретики («качественно», «профессионально»)

### Кто я

Цифровой разработчик — делаю сайты, веб-приложения и AI-агентов. Работаю напрямую, без агентства и субподрядчиков.

### Главный оффер

> **«Превращаю бизнес-задачи в цифровые продукты: сайт, приложение или AI-агент — быстро и без лишней бюрократии»**

| Я | Агентство |

| Личная ответственность за результат | Размытая ответственность между отделами |
| Три направления: веб + мобайл + AI | Обычно специализация на одном |
| Гибкость — подстраиваюсь под задачу | Продают пакеты и шаблонные решения |

### Почему мне можно доверять

- Делаю сам, а не перепродаю
- Фиксирую бюджет и сроки до начала работы
- Остаюсь на связи после сдачи проекта
- AI-экспертиза — не хайп, а конкретные инструменты под задачу

## 7. UX требования

### Навигация

- Sticky header: появляется при скролле (или всегда видима)
- Максимум 4 пункта + CTA-кнопка
- На мобильном: бургер-меню или убрать в пользу floating CTA
- Плавный скролл к якорям (`scroll-behavior: smooth`)

### Адаптивность

```
Mobile:  320px – 767px   (приоритет разработки)
Tablet:  768px – 1023px
Desktop: 1024px+
```

- Все секции корректно отображаются на всех breakpoints
- Карточки: 1 колонка на мобильном, 2–3 на десктопе
- Кнопки: минимум 44×44px (tap target)
- Текст: минимум 16px на мобильном

### Производительность

- Изображения: WebP, lazy loading, правильные размеры
- Шрифты: preconnect + display=swap
- Минимум JavaScript: только форма и анимации Intersection Observer
- Цель: PageSpeed Insights > 85 на мобильном и десктопе

### Анимации (MVP — CSS only, без GSAP)

```
1. Fade-in при скролле:
   Каждая секция появляется с opacity 0→1 + translateY(20px→0)
   Инструмент: Intersection Observer API
   Длительность: 0.5s ease-out

2. Hover на карточках:
   transform: translateY(-2px)
   box-shadow: усиление
   transition: 0.25s ease

3. Hover на кнопках:
   translateY(-1px) + box-shadow с цветом акцента
   transition: 0.2s ease

4. Floating Telegram-кнопка на мобильном:
   Фиксированная позиция bottom-right
   Появляется после скролла 200px
```

### Доступность

- Все интерактивные элементы: `:focus-visible` стили
- Контраст текста: минимум 4.5:1 (WCAG AA)
- Alt-текст для всех изображений
- Семантические HTML-теги (`<section>`, `<article>`, `<nav>`, `<main>`)

### Технические решения для спорных мест

| Задача | Решение | Альтернатива |
|---|---|---|
| Форма контакта | Formspree (бесплатный план) | Netlify Forms |
| Шрифты | `@fontsource/inter` (без Google Fonts) | Google Fonts с preconnect |
| Анимации | Intersection Observer API | CSS `@keyframes` + class toggle |
| Аккордеон FAQ | HTML `<details>/<summary>` (нативный) | JS-аккордеон |
| Floating-кнопка | CSS `position: fixed` + JS scroll listener | CSS-only с `:has()` |
| Изображения | Astro `<Image />` компонент | `<img>` с loading="lazy" |
