## Архитектура проекта

- **Стек:** Astro + Tailwind CSS
- Страницы создаются как Astro-компоненты (`.astro`)
- Компонентная структура: каждая секция — отдельный компонент в `src/components/`
- Минимизируем JavaScript: только там, где без него не обойтись (форма, интерактивные элементы)
- Сайт должен быть максимально быстрым — PageSpeed Insights > 85

---

## Дизайн

- **Стиль:** Dark + Glassmorphism
- **Фон:** `#0a0a0a` / `#111111`
- **Акцентный цвет:** Electric blue `#00d4ff` — кнопки, ссылки, hover-эффекты, акценты
- **Карточки:** `rgba(255,255,255,0.05)` + `backdrop-filter: blur(12px)` + тонкий border `rgba(255,255,255,0.08)`
- **Шрифты:** Inter или Geist (подключаем через `@fontsource` или Google Fonts)
- **Отступы между секциями:** 80–100px
- **Радиусы:** 16–24px
- **Анимации на старте:** только CSS transitions, без GSAP

---

## Вёрстка

- **Mobile-first:** сначала мобильный breakpoint, потом расширяем через `md:`, `lg:`
- Полная адаптивность для всех экранов
- Кнопки и интерактивные элементы: минимум 44×44px (tap target)
- Текст читаемый: минимум 16px на мобильном, достаточный line-height
- Контент не перегружен: белое пространство — часть дизайна
- Нет горизонтального скролла на мобильном

## Структура MVP лендинга

Порядок секций строго соблюдается:

1. **Hero** — оффер, subheadline, CTA «Обсудить проект»
2. **Обо мне** — текст, стек технологий, живое фото
3. **Что я делаю** — 3 направления: веб / приложения / AI-агенты
4. **AI-экспертиза** — объяснение AI-агентов + 3–4 сценария применения
5. **Процесс работы** — 4–5 шагов от заявки до результата
6. **Почему со мной** — преимущества vs. агентство
7. **FAQ** — 7–10 вопросов
8. **Контакты** — форма (3 поля) + Telegram + время ответа
9. **CTA** — финальный призыв перед футером

**Кейсы и отзывы — только в v2, не в MVP.**

---

## Контент

- Текст простой и дружелюбный, без корпоративных штампов
- Никакого маркетингового «воды» («профессиональный подход», «высокое качество»)
- Коротко и по делу: если можно сказать в одном предложении — не растягивать в три
- Фокус на пользе для клиента, а не на характеристиках услуги
- CTA-кнопки: не «Отправить», а «Обсудить проект» / «Написать в Telegram»

## Правила работы ИИ

- **Перед созданием новых файлов** — объяснить, что и зачем планируется создать
- **Перед установкой зависимостей** — спросить, нужна ли она, предложить альтернативу без неё
- Не устанавливать зависимости, которые дублируют то, что уже есть в Astro или Tailwind
- Соблюдать структуру проекта из [project.md](project.md)
- Следовать порядку секций из MVP — не добавлять новые без явного запроса
- При любых сомнениях в требованиях — задавать вопросы, а не принимать решения самостоятельно
