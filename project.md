## 1. Что это за проект

Лендинг-портфолио цифрового мастера — одностраничный сайт для привлечения клиентов на услуги веб-разработки, мобильных приложений и AI-агентов. Сайт выступает главным инструментом личного бренда и первой точкой контакта с потенциальными клиентами. Цель — конвертировать посетителей в заявки через чёткое позиционирование, кейсы с результатами и прямые каналы связи.

---

## 2. Для кого

**Основная аудитория:** стартапы и продуктовые команды, которым нужен технический партнёр для запуска или развития цифрового продукта.

**Вторичная аудитория:** малый бизнес, которому нужна автоматизация через AI-агентов или быстрый старт в онлайн.

**Портрет клиента:**
- Технически грамотен или хочет разобраться
- Ценит скорость, прямую коммуникацию и личную ответственность
- Устал от агентств с бюрократией и размытыми сроками
- Интересуется AI-инструментами, но не знает, как их применить

---

## 3. Проблема, которую решает

Стартапы и продуктовые команды теряют время на поиск надёжного технического исполнителя. Агентства медленные и дорогие, биржи рискованные. Нет понятного места, где можно быстро оценить компетенции, посмотреть реальные результаты и выйти на прямой контакт с человеком, который сам делает работу.

---

## 4. Основное позиционирование

> **«Превращаю бизнес-задачи в цифровые продукты: сайт, приложение или AI-агент — быстро и без лишней бюрократии»**

**Дифференциаторы:**
- Три направления в одном: веб + мобайл + AI-агенты
- Личная ответственность и прямой контакт (не агентство)
- Специализация на AI-агентах — растущая ниша, низкая конкуренция
- Фокус на результате, а не на процессе

## 7. MVP — функции на старте

Минимальный работающий лендинг, готовый принимать заявки:

| # | Секция | Содержимое |
|---|---|---|
| 1 | **Hero** | Оффер, subheadline, CTA «Обсудить проект» |
| 2 | **Обо мне** | Текст о себе, стек технологий, живое фото |
| 3 | **Что я делаю** | 3 направления: веб / приложения / AI-агенты |
| 4 | **AI-экспертиза** | Объяснение AI-агентов простым языком + 3–4 сценария применения |
| 5 | **Процесс работы** | 4–5 шагов: от заявки до результата |
| 6 | **Почему со мной** | Преимущества vs. агентство: скорость, личная ответственность, прямой контакт |
| 7 | **FAQ** | 7–10 вопросов с закрытием ключевых возражений |
| 8 | **Контакты** | Форма (3 поля) + Telegram + время ответа |
| — | **CTA** | Финальный призыв к действию перед футером |

**Технические обязательства MVP:**
- [ ] Мобильная версия — полноценная, mobile-first
- [ ] SEO-основа — meta title, description, Open Graph

| Тип | Примеры | Чем отличаемся |
|---|---|---|
| Авторские 3D-портфолио | Bruno Simon, Jordan Breton, Samsy Ninja | У них WOW-эффект, но нет фокуса на конверсии для бизнеса |

**Наше преимущество:** сочетание технической экспертизы (включая AI) + личного бренда + ориентации на результат для бизнеса.

## 10. Технологии

### Ядро
| Слой | Технология | Почему |
|---|---|---|
| Фреймворк | **Astro** | Минимум JS, максимум производительности, идеален для лендингов |
| Стили | **Tailwind CSS** | Утилитарный подход, быстро, консистентно |
| Анимации (MVP) | **CSS transitions** | Легко, без зависимостей |
| Анимации (v2) | **GSAP + ScrollTrigger** | Индустриальный стандарт для scroll-анимаций |
| Деплой | **Vercel** или **Netlify** | Бесплатно, быстро, CDN из коробки |
| Форма | **Formspree** или **Netlify Forms** | Без бэкенда, просто |

### Возможные добавления (v2)
- **Three.js / Spline** — 3D-элемент в Hero
- **Lenis** — smooth scroll
- **SplitType.js** — кинетическая типографика

## 11. Стиль дизайна

**Направление: Dark + Glassmorphism**

| Параметр | Значение |
|---|---|
| Фон | `#0a0a0a` / `#111111` |
| Акцентный цвет | Electric blue `#00d4ff` — кнопки, ссылки, hover-эффекты, акценты |
| Карточки | `rgba(255,255,255,0.05)` + `backdrop-filter: blur(12px)` |
| Типографика | Крупная, белая, контрастная. Inter / Geist / Space Grotesk |
| Иконки | Минималистичные, линейные |
| Текстура | Grain overlay (опционально, даёт «человечность») |
| Отступы | Щедрые — 80–100px между секциями |
| Радиусы | 16–24px (современный, не агрессивный) |

**Анимации на старте:**
- Плавные hover-эффекты на карточках
- Fade-in при скролле (CSS `@keyframes` или Intersection Observer)
- Кастомный курсор (опционально)

---

## 12. Чего не будет на старте

- Логотипы клиентов (нет готовых материалов)
- Видеоотзывы и текстовые отзывы с фото (нет клиентов пока)
- Блог
- 3D / WebGL эффекты (откладываем в v2)
- Автоплей видео, попапы при входе — **никогда**
- Форма с больше чем 3 полями
- Цены «по запросу» без ориентиров (нужно хотя бы указать диапазон)
- Навигация с 5+ пунктами (максимум 4 якоря)

## 13. Первые шаги разработки

```
Шаг 1: Инициализация проекта
  └── npx create astro@latest
  └── Подключить Tailwind CSS
  └── Настроить деплой на Vercel

Шаг 2: Дизайн-система
  └── Цветовая палитра в tailwind.config.js
  └── Типографика: шрифт + размерная шкала
  └── Базовые компоненты: Button, Card, Section, Badge

Шаг 3: Контент (параллельно с разработкой)
  └── Написать оффер (H1 + subheadline)
  └── Написать текст «О себе» + выбрать фото
  └── Описать 3 направления услуг
  └── Подготовить блок AI-экспертизы (3–4 сценария)
  └── Сформулировать преимущества «Почему со мной»
  └── Составить FAQ (7–10 вопросов)

Шаг 4: Верстка секций (по приоритету MVP)
  1. Hero
  2. Обо мне
  3. Что я делаю
  4. AI-экспертиза
  5. Процесс работы
  6. Почему со мной
  7. FAQ
  8. Контакты + CTA

Шаг 5: Мобильная версия и производительность
  └── Проверить все breakpoints
  └── PageSpeed Insights > 85
  └── Проверить tap-targets

Шаг 6: SEO и запуск
  └── meta title, description, OG-теги
  └── sitemap.xml
  └── Деплой и финальная проверка
```
